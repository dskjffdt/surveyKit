## 一、简历项目栏（推荐 · 分模块写法，直接复制）

**SurveyKit | 全栈在线问卷系统**  
独立开发 · 2025.XX – 2026.XX  
GitHub：https://github.com/dskjffdt/surveyKit · 在线演示：你的 Demo 地址

**项目概述：**  
基于 Web 的全栈在线问卷系统，覆盖问卷创建、编辑、预览、发布、公开填答与结果分析，支持单选、多选、填空、评分四种题型。前端 React + TypeScript + React Query，后端 Express + MySQL + JWT httpOnly Cookie 鉴权，实现多角色权限隔离与完整业务闭环。

**技术栈：** React、TypeScript、TanStack React Query、Zustand、React Router、Node.js、Express、MySQL、Vite

**项目亮点：**

- **多题型类型建模**：基于 TypeScript 联合类型定义四种题型及差异化答案结构，通过类型收窄统一编辑、填答、统计三端处理逻辑，校验与聚合统计复用同一套类型体系。

- **全栈业务闭环**：独立完成 REST API 与 MySQL 数据模型，打通问卷 CRUD、发布/撤回、公开填答、答卷回收、统计分析与 CSV 导出；前后端均做角色权限过滤。

- **问卷编辑与预览**：题目组件编辑/填答双模式复用，支持拖拽排序；提供编辑页弹窗预览、草稿新窗口预览，预览模式可交互但不提交答卷。

- **服务端状态与性能**：React Query 管理问卷/答卷缓存与变更；路由懒加载 + Vendor 分包优化首屏；PageGate 延迟/最短展示 loading，减少页面切换闪屏。

- **鉴权与安全**：JWT 存 httpOnly Cookie，Express 中间件鉴权 + 前端路由守卫；区分 admin（全平台只读）与 creator（问卷 CRUD）操作范围。

**项目成果：**

- 独立完成问卷配置 → 预览 → 发布投放 → 答卷回收 → 统计分析 → 数据导出全流程
- 支持 4 种题型、2 种角色、6 个核心页面（含预览页）
- 生产构建入口 chunk 约 7KB gzip，前端可部署 Vercel

## 四、项目亮点（面试官问「有什么亮点」）

我觉得有三个比较实在的亮点：

**第一，业务闭环完整。** 不是只做编辑器或填答页，而是从创建、预览、发布、填答到统计导出全部打通，能完整演示。

**第二，类型设计有针对性。** 四种题型答案结构不同，我用联合类型把编辑、填答、统计三处逻辑统一起来，避免后期维护时类型混乱。

**第三，全栈 + 体验兼顾。** 包括拖拽排序、双模式渲染、草稿预览、React Query 缓存、路由分包，不只是基础 CRUD 页面。

## 五、难点回答（必背）

难点主要在两个地方：

**第一个是「多题型场景下的数据一致性」。** 同一个题型会同时出现在编辑、填答、统计三个环节，答案类型各不相同。我用联合类型和分支收窄把三处逻辑统一起来，渲染层用双模式复用避免重复开发。

**第二个是「业务状态流转的完整性」。** 比如发布态控制、校验、重复提交、未保存提醒、退出登录闪屏处理，这些细节串起来，项目才真正可用。

## 六、常见追问速答

**为什么用 React Query？**  
问卷和答卷都是服务端数据，需要缓存、失效、loading 状态；比全部放 Zustand 更合适，职责也更清晰。

**为什么还用 Zustand？**  
只管登录会话（user、logout），和 React Query 分工明确。

**为什么 JWT 放 Cookie 而不是 localStorage？**  
httpOnly Cookie 前端 JS 读不到，能降低 XSS 偷 token 的风险。

**admin 和 creator 权限区别？**  
admin 能查看全平台问卷与统计；creator 只能 CRUD 自己创建的问卷。前后端都做了过滤。

**统计图表怎么做的？**  
后端聚合数据，前端用 CSS 柱状图展示，不引入重型图表库，减轻包体积。

**预览功能怎么做的？**  
编辑页弹窗用本地缓存数据即时预览；`/preview/:id` 走鉴权接口支持草稿预览；预览模式隐藏提交，真实填答走 `/fill/:id`。

**拖拽排序怎么实现的？**  
题目左侧拖拽手柄，松手后更新题目数组顺序并保存到后端。

**演示账号？**  
admin / admin123（管理员），creator / creator123（创建者）。

## 七、演示顺序（面试 / 答辩用）

1. 首页看答卷概览卡片  
2. 用 creator 登录，创建问卷  
3. 编辑页添加多种题型，点「预览」看弹窗效果  
4. 草稿状态打开新窗口预览（`/preview/:id`）  
5. 发布问卷，复制填答链接  
6. 打开填答页（无需登录）填写并提交  
7. 统计页查看柱状图统计、导出 CSV  

全程约 3–4 分钟。
